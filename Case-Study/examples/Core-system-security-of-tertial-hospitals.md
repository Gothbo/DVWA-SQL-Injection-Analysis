# 售前案例学习笔记：某三甲医院核心业务系统安全加固
> *本案例基于公开安全事件与行业通行做法，结合我个人对售前的理解整理而成。*
> *文末附有我的复盘思考。*
---

| 字段 | 内容 |
|------|------|
| 客户名称 | 某三甲综合医院（脱敏） |
| 所属行业 | 医疗 |

| 案例日期 | 2026年 |

---

## 二、客户背景

- **现状描述**：HIS系统近期遭遇勒索软件攻击，部分门诊挂号服务中断2小时。医院信息科共5人，无专职安全人员。现有安全设备包括防火墙和杀毒软件，但缺乏主动防御能力。卫健委要求加强医疗信息系统安全防护。
- **已有安全投入**：基础防火墙、终端杀毒软件、网络隔离（内外网分离），但缺少入侵检测、数据备份和应急响应能力。
- **行业监管要求**：《医疗卫生机构网络安全管理办法》《电子病历系统安全规范》《个人信息保护法》（患者隐私保护），等保三级要求。

---

## 三、我的思考

- **打动客户的点**："业务不中断"——医院最怕安全方案影响HIS系统，旁路部署和热更新是成交关键
- **简化表达**：EDR防勒索 + 备份快恢复 + 专家远程支持，三件事讲清楚
- **可迁移场景**：妇幼保健院、专科医院、医联体/医共体
- **竞品攻击点**："EDR大家都做，你们有什么区别？" 防守："我们的核心优势是医疗行业适配——HIS系统兼容性测试通过，不影响业务运行，这是通用方案做不到的"
- **学到的技巧**：医疗行业要懂业务流程——挂号、就诊、检验、取药，安全方案要嵌入业务流而不是打断业务流


---

## 四、痛点分析

**一句话痛点：** 勒索攻击导致挂号服务中断2小时，信息科5人没专职安全人员，卫健委要求加强防护。

| 表面诉求 | 深层痛点 | 翻译为技术需求 |
|----------|---------|--------------|
| "HIS系统遭遇勒索攻击，挂号服务中断2小时" | 核心业务系统缺乏主动防御，被攻击后无法快速恢复 | 需要终端EDR防护 + 核心系统数据备份 + 应急响应预案 |
| "信息科5人，没有专职安全人员" | 安全能力与业务重要性严重不匹配，没人懂安全 | 需要轻量级安全方案 + 厂商安全专家远程支持 |
| "卫健委要求加强医疗信息系统安全防护" | 合规压力 + 患者隐私保护要求日益严格 | 需要符合等保三级的完整防护体系 |
| "内外网物理隔离了但还是被攻击" | 隔离不是万能的，内部U盘、移动设备、第三方接口都是入口 | 需要终端管控 + 接口安全 + 数据防泄露 |

---

## 五、解决方案

| 产品/服务 | 解决什么问题 | 为什么选这个 |
|-----------|------------|-------------|
| EDR终端防护（全院部署） | 勒索软件、恶意程序主动防御 | 替代传统杀毒，解决"内外网隔离了还被攻击"的问题 |
| 核心业务数据备份方案 | HIS/LIS数据实时备份，勒索攻击后快速恢复 | 解决"服务中断2小时"的痛点 |
| 接口安全网关 | 医保接口、检验结果查询接口安全防护 | 防止第三方接口成为攻击入口 |
| 安全专家应急响应服务 | 被攻击后快速定位、隔离、恢复 | 弥补信息科无人懂安全的能力缺口 |
| 等保合规咨询 + 整改方案 | 满足卫健委和等保三级要求 | 合规驱动，一次投入满足多项要求 |

**方案亮点：**

- **业务不中断**——所有安全设备旁路部署或热更新，不影响HIS系统7×24运行
- **防勒索为核心**——EDR + 数据备份双保险，即使被加密也能快速恢复
- **轻量化设计**——针对5人信息科，选择运维简单的方案，不增加人力负担
- **合规+安全双驱动**——方案同时满足等保三级和卫健委要求
